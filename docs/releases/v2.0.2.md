# ries-rs v2.0.2

Patch release correcting turbo rank-one parity for streaming-sized searches.

## Bug fixes

- `--turbo` now mirrors serial generation semantics before parallel matching.
  Small searches continue to use the same target-local LHS deduplication as the
  serial batch path, while larger searches that would stream serially preserve
  the raw LHS stream before splitting the match/Newton work across workers.
- This fixes a level-3 miss for target `2.506314`, where batch LHS deduplication
  could keep a lower-complexity degenerate representative for the same
  target-local `(value, derivative)` bucket and discard the serial best match
  `cospi(e^tanpi(x)) = 1` before turbo matching.
- Added regression coverage for both the small batch-generation turbo path and
  the expensive level-3 flat-exact `2.506314` case.

## Compatibility

- No public API changes.
- Turbo retains its documented alternative contract: the single best match
  matches serial search, while lower-ranked tail results may differ.

## Distribution notes

- CLI archives are provided for Linux x86_64, macOS x86_64 and arm64, and
  Windows x86_64.
- The release includes browser, Node.js, and bundler WASM packages.
- PyPI artifacts include stable-ABI wheels and a source distribution.
