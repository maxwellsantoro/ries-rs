# ries-rs v2.0.1

Patch release correcting order-dependent stopping behavior in turbo and
parallel-generation searches.

## Bug fixes

- `--turbo` no longer applies `--stop-at-exact` or `--stop-below` independently
  inside parallel matching bands. These configurations now use the canonical
  serial pipeline so an early worker result cannot hide the globally best
  match.
- The ordinary parallel-generation path uses the same fallback for first-hit
  stopping options, preserving its byte-identical serial-search contract.
- Turbo CLI output is canonically sorted before presentation, ensuring the
  documented best match appears at rank one even when a serial fallback
  prioritizes a qualifying stop match internally.

## Compatibility

- No public API changes.
- Exhaustive turbo searches remain parallel and retain the v2.0.0 result
  contract. Only order-dependent first-hit configurations fall back to serial
  execution.

## Distribution notes

- CLI archives are provided for Linux x86_64, macOS x86_64 and arm64, and
  Windows x86_64.
- The release includes browser, Node.js, and bundler WASM packages.
- PyPI artifacts include stable-ABI wheels and a source distribution.
- No Zenodo version DOI is recorded until the v2.0.1 archive has completed. The
  project concept DOI remains `10.5281/zenodo.19101924`.
